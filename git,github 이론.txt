Git, Github :: 형상 관리 프로 그램

로컬 저장소 :: 각 컴퓨터에서만 접근할 수 있는 저장소
원격 저장소 :: 서버를 통해서 마련해둔 저장소(ex : Git, Github)


Github는 Git을 좀 더 편리하게 사용할 수 있게끔 하는 역할

<Git Bash 명령어>
git init :: git과 연결할 준비 (새롭게 만든 폴더에 가장 먼저 적용)
git clone :: 로컬저장소-원격저장소를 연결하는 명령어(원격 저장소 복제)
git remote add [변수명] [원격지주소] 
          :: 원격지주소의 값을 변수에 집어넣어 치환
            이 문법을 사용하면, 이 변수 하나로 원격지주소 사용 가능

commit :: "반영하다" / 푸쉬 이전에 작성, 변경사항 기록
push :: "밀다, 밀어넣다" / 로컬->원격 업로드
pull :: "당기다/가져오다" / 원격->로컬 다운로드
branch :: "나뭇가지,분기"

<push 과정>
- git config user.name "사용자이름"
- git config user.email "사용자이메일"
- git add [작업할 파일 지정]
            . 또는 *을 넣으면, 폴더 전체에 대해서 작업을 한다
            (git add . / git add *  띄어쓰기 필수)
- git commit -m "변경사항메시지" 
            변경사항 반영(+변경사항에 대한 메시지 기록)
- git push [원격지주소] [branch명]
             로컬 -> 원격저장소로 업로드하는 과정
<pull 과정>
- git pull [원격지주소] [branch명]
             원격 -> 로컬저장소로 업로드하는 과정
